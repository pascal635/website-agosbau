## Schema-Daten (für den Bau, Phase 6)

> JSON-LD im Bau setzen, Markup muss den sichtbaren Text spiegeln (siehe `references/playbooks/schema-structured-data.md`). `AggregateRating` an `LocalBusiness` der Startseite/Kontakt; einzelne `Review`-Objekte dort, wo das Testimonial sichtbar ist.

**AggregateRating (an LocalBusiness):**
```json
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "AGOS BAU",
  "image": "https://agosbau.de/...",
  "address": {"@type": "PostalAddress","streetAddress": "Auf den Mehren 45","postalCode": "28832","addressLocality": "Achim","addressCountry": "DE"},
  "telephone": "+49 4202 5116131",
  "url": "https://agosbau.de",
  "aggregateRating": {"@type": "AggregateRating","ratingValue": "5.0","reviewCount": "25","bestRating": "5"}
}
```

**Beispiel Review-Objekt (je sichtbarem Testimonial eines):**
```json
{
  "@type": "Review",
  "author": {"@type": "Person","name": "Christian Heinzl"},
  "datePublished": "2025-11",
  "reviewRating": {"@type": "Rating","ratingValue": "5","bestRating": "5"},
  "reviewBody": "Wir sind mit der Arbeit von AGOS Bau wirklich zu 100 Prozent zufrieden. Von der Beratung bis zur Ausführung hat alles wunderbar geklappt. Unser Badezimmer sieht wirklich klasse jetzt aus, mit den großen Fliesen."
}
```
*Datumsangaben: aus „vor X Monaten/Jahren" beim Bau auf konkrete Monate umrechnen (Bezug: Juni 2026). Nur sichtbar gezeigte Reviews ins Markup (Google-Richtlinie).*
